B형 문제는 보통 아이디어를 활용한 최적화 문제가 많이 출제 된다

1. 문제에서 수상하다고 생각되는 부분을 메모한다

2. 최대한 단순하게 구현할 수 있는 방법을 생각한다

3. 함수 호출 횟수가 주어질 때, 시간 복잡도를 계산하여 절대 불가능한 것을 찾아야 한다
    - 전체의 함수 호출 횟수가 주어진다면, 하나의 method가 전부 호출될 때도 고려해야 한다.

4. 느려지게 되는 부분을 찾아서 최적화 할 수 있는 것을 생각한다
    - 값을 찾으려는 시점에 느리다면, 담는 시점에 다르게 저장 => 자료구조
    - 담는게 느리다면 편하게 담고, 나중에 찾을 때 처리 => lazy
